Git 是什么？

存档系统与版本管理

Git 就像游戏里的存档系统或写论文时的版本备份（如 v1.doc、vfinal.doc）。它帮你优雅地追踪文件的每一次变化，让你随时可以回到过去的状态。

本地运行，追踪文件的每一次变化

版本追踪 · 本地存档 · 分支管理

云端存储代码，提供团队协作平台

代码托管 · 在线协作 · Pull Request

Commit — 存档点

快照 (Snapshot)

每次做一次 Commit，就像给当时的代码拍一张照片。通过这些快照，你可以组成完整的代码演进历史。

Branch — 平行宇宙

隔离实验环境

main 分支是稳定运行的正式版。创建分支就像分出一个平行宇宙，在里面开发新功能不会影响到主时间线。

Merge — 合并

时间线收束

当新分支的功能开发完毕，通过 Merge (合并) 可以将平行宇宙中的改动带回到 main 主干中。

完整工作流程

1. Clone (克隆)

将远程代码库下载到本地

git clone <url>

2. Branch (分支)

创建隔离的开发环境

git checkout -b feature

3. Commit (提交)

保存代码快照

git commit -m "update"

4. Push (推送)

上传本地分支到云端

git push origin feature

5. Pull Request

在 GitHub 发起合并请求，进行审查

6. Merge (合并)

审查通过后并入主干
